# HTML — Company-Wise Interview Patterns

> Reflects general patterns reported for each company category, not confirmed question banks.

## Service-Based Companies (TCS, Infosys, Wipro, Cognizant, Accenture)
- **Focus**: Fundamentals — tag semantics, forms, tables, basic accessibility.
- **Style**: MCQ round + short coding/markup task, then a technical interview.
- **Typical asks**: Difference between div/span, semantic tags, form GET vs POST, build a simple page layout live.

## Product-Based Companies (Amazon, Flipkart, Microsoft, Swiggy, Zomato, Myntra)
- **Focus**: Real UI build tasks combined with CSS/JS, performance and accessibility awareness.
- **Style**: Live coding — "build this UI from a Figma/design mock" — often combined with CSS/JS in one round.
- **Typical asks**: Build a responsive card/product-listing layout, explain Core Web Vitals impact of your markup choices, accessibility of a custom component you built.

## Design-Heavy / Consumer Product Companies (fintech apps, D2C brands, media companies)
- **Focus**: Pixel-perfect implementation, cross-browser consistency, semantic + accessible markup.
- **Typical asks**: Implement a given design exactly, explain how you'd support older browsers, discuss `<picture>`/responsive images strategy.

## Startups
- **Focus**: Practical speed — can you turn a design into working, responsive, reasonably accessible HTML/CSS quickly.
- **Typical asks**: Take-home: build a landing page from a Figma link; live pairing on a small feature.

## Testing/QA-Adjacent Roles
- **Focus**: Understanding DOM structure well enough to write reliable selectors (for Selenium/Cypress/Playwright).
- **Typical asks**: How would you write a robust, stable selector for this element; explain how markup changes could break automated tests.

## General Pattern Across All Companies
1. **Round 1** — Online assessment/portfolio review or small take-home build.
2. **Round 2** — Live coding: build/fix a UI component from a mock or bug report.
3. **Round 3** — Deeper technical (often merged with CSS/JS): performance, accessibility, responsive design reasoning.
4. **Round 4** — Managerial round.
5. **Round 5** — HR round.

## What Interviewers Commonly Evaluate
- Semantic correctness (not just "does it look right")
- Accessibility awareness without being prompted
- Responsive/mobile-first thinking
- Clean, valid, well-structured markup
- Performance awareness (image loading, script placement)
